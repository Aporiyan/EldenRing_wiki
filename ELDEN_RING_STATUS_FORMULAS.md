## 2. 攻击力（AR）计算

### 2.1 总公式（每类伤害独立计算）

```
AR_最终 = 基础伤害 × 强化倍率 + ∑( 各属性补正伤害 )
```

每项**属性补正伤害**公式：

```
补正伤害 = 基础伤害 × 伤害占比% × (修正值 × 修正倍率 / 100) × (饱和系数 / 100)
```

其中：
- **基础伤害**：`EquipParamWeapon → attackBasePhysics / attackBaseMagic` 等
- **强化倍率**：`ReinforceParamWeapon → physicsAtkRate / magicAtkRate`（+25 时为 2.45，+10 为 2.20）
- **修正值** `correctStat`：武器本身存储的数值（如 correctStrength=35）
- **修正倍率** `correctRate`：强化等级带来的乘数（+25 通常为 1.5x）
- **饱和系数**：来自 `CalcCorrectGraph`，决定软上限的曲线

### 2.2 CalcCorrectGraph（饱和曲线）

这是决定"多少属性值 → 多大百分比"的核心查找表。

```
out = stageMaxGrowVal_N + ((input - stageMaxVal_N) / (stageMaxVal_{N+1} - stageMaxVal_N)) × (stageMaxGrowVal_{N+1} - stageMaxGrowVal_N)
```

**物理类武器（CalcCorrectGraph_ID = 0，标准/厚重/锋利/优质/神秘）**

| Stat | 饱和值% |
|------|---------|
| 1 | 0 |
| 10 | 15 |
| 20 | 35 |
| 30 | 46.5 |
| 40 | 57.5 |
| 50 | 67.5 |
| 60 | 75 |
| 70 | 82.5 |
| 80 | 90 |
| 90 | 92.5 |
| 99 | 96 |

> 物理曲线在 10→20 增幅最大；20→40 逐步放缓；50→60 再放缓；80 之后近乎停滞。

**非物理类武器（CalcCorrectGraph_ID = 4，魔/火/雷/圣/血/毒/冰/等）**

| Stat | 饱和值% |
|------|---------|
| 1 | 0 |
| 10 | 19 |
| 20 | 40 |
| 30 | 53 |
| 40 | 66.5 |
| 50 | 80 |
| 60 | 85 |
| 70 | 90 |
| 80 | 95 |
| 90 | 97.5 |
| 99 | 100 |

> 非物理曲线更"线性"，50 点 ≈ 80%，相比物理的 67.5% 效率更高。

### 2.3 共鸣率（Scaling Value）与字母等级

```
补正后的修正值 = 武器correctStat × 强化correctRate
```

| 字母等级 | 修正值范围 |
|----------|-----------|
| S | ≥ 175 |
| A | 140 ≤ x ≤ 175 |
| B | 90 ≤ x < 140 |
| C | 60 ≤ x < 90 |
| D | 25 ≤ x < 60 |
| E | x < 25 |

### 2.4 双手共持

```
有效STR = 实际STR × 1.5（上限 99）
```

仅对 STR 的物理补正生效；不影响 DEX/INT/FTH/ARC 补正，也不要求满足力气不足时为灰色。

### 2.5 具体计算示例

**+25 标准匕首，STR 20，DEX 30**

```
attackBasePhysics = 74
physicsAtkRate(+25) = 2.45
基础伤害 = 74 × 2.45 = 181.3

correctStrength = 35, correctStrengthRate(+25) = 1.5 → 修改后 = 52.5 (D级)
correctAgility = 65, correctAgilityRate(+25) = 1.5 → 修改后 = 97.5 (B级)

CalcCorrectGraph ID=0:
  STR 20: 饱和值 = 35%
  DEX 30: 饱和值 = 46.5%

STR补正 = 74 × 1.0 × (52.5 / 100) × (35 / 100) = 13.60
DEX补正 = 74 × 1.0 × (97.5 / 100) × (46.5 / 100) = 33.55

AR = 181 + 14 + 34 = 229（进一取整）
```

> 注：AR 显示的 "+xxx" 为所有补正合计，不含 Motion Value（动作值在攻击时而非状态面板计算）。

## 3. 防御力（Defense）

### 3.1 等级基础防御（所有 8 类伤害共用）

```
等级基础 = 按RL查表
```

| 条件 | 公式 |
|------|------|
| RL < 72 | 40 + (RL + 78) ÷ 2.483 |
| 72 ≤ RL < 92 | 29 + RL |
| 92 ≤ RL < 161 | 120 + (RL − 91) ÷ 4.667 |
| 161 ≤ RL | 135 + (RL − 161) ÷ 27.6 |

```
总防御 = 等级基础 + 属性防御补正
```

## 4. 属性防御

每类的**总防御 = 等级基础（同 3.1）+ 对应属性补正**

### 4.1 Physical Defense（物理）— STR

| STR 范围 | 防御补正 |
|----------|----------|
| < 30 | STR ÷ 3 |
| 30 ≤ x < 40 | 10 + (x − 30) ÷ 2 |
| 40 ≤ x < 60 | 15 + (x − 40) ÷ 1.333 |
| ≥ 60 | 30 + (x − 60) ÷ 3.9 |

### 4.2 Magic Defense（魔力）— INT

| INT 范围 | 防御补正 |
|----------|----------|
| < 20 | INT × 2 |
| 20 ≤ x < 35 | 40 + (x − 20) ÷ 1.5 |
| 35 ≤ x < 60 | 50 + (x − 35) ÷ 2.5 |
| ≥ 60 | 60 + (x − 60) ÷ 3.9 |

### 4.3 Fire Defense（火）— VIG

| VIG 范围 | 防御补正 |
|----------|----------|
| < 30 | VIG ÷ 1.5 |
| 30 ≤ x < 40 | 20 + (x − 30) × 2 |
| 40 ≤ x < 60 | 等于 Vigor 值 |
| ≥ 60 | 60 + (x − 60) ÷ 3.9 |

### 4.4 Lightning Defense（雷）

**无属性加成**，仅依赖等级基础（3.1）。

### 4.5 Holy Defense（圣）— ARC

| ARC 范围 | 每级增长 |
|----------|----------|
| 1–20 | +2.0 / 级 |
| 21–35 | +0.66 / 级 |
| 36–60 | +0.4 / 级 |
| 61–99 | +0.25 / 级 |

> 示例：ARC 50 → 圣防御补正 = 20×2 + 15×0.66 + 15×0.4 = 40 + 9.9 + 6.0 = 55.9

### 4.6 四类物理防御细节（打击/斩击/突刺）

状态面板显示 Strike / Slash / Pierce 三个独立防御值。

```
打击防御 = 物理防御 + 装备盾牌补正（不同盾牌不同比率）
斩击防御 = 物理防御 + 装备盾牌补正
突刺防御 = 物理防御 + 装备盾牌补正
```

> 公式与 4.1 完全相同，但四者共享同一个物理防御数值。差异仅来自装备/盾牌的独立补正。

## 10. 观察力（Item Discovery）

```
观察力 = 100 + ARC + 装备/道具加成
```

- 基础值固定为 **100**
- 每 1 级 ARC → +1 观察力（线性，无递减）
- 银粪金龟护符：+75
- 银色泪滴面具：+8 ARC
- 白金之子面具：+4 ARC
- 银色金鸡爪（消耗品）：+50（3分钟）

**物品掉落最终概率**（源自数据挖掘精确公式）：

```
W = P × 10 × (D / 100)
有效概率% = (W × 100) / (1000 + W − P × 10)
```
其中 P = 基础概率（百分比），D = 观察力

**近似简式**（与原公式 1% 内误差）：
```
有效概率 ≈ 基础概率 × (观察力 / 100)
```

## 11. 记忆空格（Memory Slots）

**艾尔登法环中记忆空格不随 MND 等级增加。**

- 基础：**2 格**（所有职业）
- 记忆石（Memory Stone）：每颗 +1 格，最多 **8 颗** → 10 格
- 诺克史黛拉之月（护符）：+2 格
- 理论最大值：**12 格**（8 记忆石 + 月护符）

> 这是一个重要改变——不像《黑暗之魂》系列通过集中力（Attunement）增加法术栏。

## 12. 韧性（Poise）

```
韧性 = ∑ 头盔韧性 + 胸甲韧性 + 手套韧性 + 护腿韧性 + 护符/效果加成
```

- 纯为装备数值叠加
- 不受任何属性等级影响
- 常见范围：0（裸装）– 约 130+（大山羊套 + 硬韧护符）

## 13. 法术触媒补正

### 13.1 法杖补正（Sorcery Scaling）

与武器 AR 使用完全相同的系统：
```
法杖补正 = (法杖基础补正 × 强化倍率) + ∑(属性补正)
```

属性补正同样使用 `CalcCorrectGraph`，但法杖有各自的 ID（通常 ID=6 或 8 等）。

- 普通法杖仅从 INT 获得补正
- 特殊法杖：
  - 亚人女王杖：高基础低 INT 倍率
  - 格密尔辉石杖：INT + FTH
  - 死亡子杖：INT + FTH（最高潜力）
  - 白金杖：INT + ARC
  - 罪人杖：仅 FTH

### 13.2 圣印补正（Incantation Scaling）

同一套系统，使用不同 CalcCorrectGraph：
- 一般圣印：仅 FTH
- 爪痕圣印：STR + FTH
- 龙飨圣印：ARC + FTH
- 黄金律法圣印：INT + FTH
- 癫火圣印：STR + DEX + INT + FTH（全属性）

> 补正值直接作为百分比乘数应用于法术的基础伤害。

## 14. 伤害减免（Damage Negation）

### 14.1 装备减免
```
装备减免 = 所有装备部位的减免叠加（非加和，而是叠乘）
总减免% = 1 − ∏(1 − 各部位减免值%)
```
例如头盔 5%，胸甲 15%，护手 3%，护腿 8%，护符 10%：
```
总减免 = 1 − (1−0.05)(1−0.15)(1−0.03)(1−0.08)(1−0.10) = 1 − 0.641 = 35.9%
```

### 14.2 实际伤害计算

```
攻击比 = (攻击力 × 动作值) / 防御力
```

| 攻击比范围 | 防御倍率 |
|-----------|---------|
| < 0.125 | 10% |
| 0.125 ≤ x < 1 | 10% + (x − 0.125)² ÷ 2.552 |
| 1 ≤ x < 2.5 | 70% − (2.5 − x)² ÷ 7.5 |
| 2.5 ≤ x < 8 | 90% − (8 − x)² ÷ 151.25 |
| ≥ 8 | 90% |

```
最终伤害 = (100% − 减免%) × 防御倍率 × 攻击力 × 动作值
```
